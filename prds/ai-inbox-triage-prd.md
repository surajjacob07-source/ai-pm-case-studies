# PRD: AI Inbox Triage

## Objective
Help busy founders and operators process email faster by identifying urgency, drafting suggested responses, and bundling messages by action type.

## Target User
A high-context user handling too many inbound threads daily.

## User Problem
Email overload causes slow responses, missed priorities, and constant context switching.

## Product Hypothesis
If the inbox is pre-sorted into actionable clusters with reliable summaries and suggested next steps, users will process mail faster with less cognitive load.

## MVP Features
- Urgent vs non-urgent classification
- Thread summaries
- Suggested reply drafts
- Action labels: respond, delegate, read later, archive

## Success Metrics
- Reduction in time-to-inbox-zero
- % of suggestions accepted or lightly edited
- Increase in response time on urgent threads

## Risks
- Bad prioritization damages trust
- Privacy concerns around sensitive mail
- Over-automation may create embarrassing replies

## Open Questions
- Should drafts be shown inline or in a review queue?
- How much user customization is needed for prioritization rules?
- What confidence threshold should trigger automation suggestions?
